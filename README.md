# Small Language Model (SLM) from Scratch

This repository contains the code, dataset, and documentation for building a Small Language Model (SLM) from scratch. The project focuses on processing a corpus of medical text (PubMed abstracts), tokenizing the data, training a custom transformer-based language model, and evaluating its generated text.

## Repository Contents

Below is a detailed explanation of each file and directory included in this repository:

### 📄 Documentation & Notebooks
- **`Build SLM.pdf`**: The project assignment brief and documentation. It outlines the requirements, objectives, and steps for building the Small Language Model.
- **`Vizuara_AI_Labs_Small_Language_Model_Scratch_Final.ipynb`**: The primary Jupyter Notebook. This file contains the complete end-to-end code for the project, including the transformer architecture definition, data loading, model training loop, evaluation, and inference.

### ⚙️ Scripts & Data Processing
- **`dataset.py`**: A Python script responsible for data preprocessing. It reads the raw data from `pubmed_dataset/pubmed_abstracts.csv`, extracts the abstract text, removes URLs, cleans up whitespace, filters out invalid or short abstracts, and compiles the final clean text into `pubmed_corpus.txt`.

### 📂 Datasets & Corpus
- **`pubmed_dataset/`**: A directory containing the original, raw dataset.
  - **`pubmed_abstracts.csv`**: The raw CSV file containing paper titles, links, and abstracts.
- **`pubmed_corpus.txt`**: The intermediate processed text file generated by `dataset.py`. This file contains all the cleaned medical abstracts combined into a single text corpus, ready to be tokenized.
- **`train.bin`**: The tokenized binary training dataset. This contains the integer representations of the tokens that the model uses to learn during the training phase.
- **`validation.bin`**: The tokenized binary validation dataset. This is a holdout set used to evaluate the model's loss and prevent overfitting during training.

### 📊 Results & Outputs
- **`output_50000.txt`**: A log of sample texts generated by the model after 50,000 training steps. It demonstrates the model's ability to generate coherent text based on specific medical prompts, such as "The effects of COVID-19 on", "Patients diagnosed with type 2 diabetes", and "MRI imaging results suggest".

## How to Run

1. **Data Preparation**: If you need to recreate the text corpus from the raw CSV, run the `dataset.py` script:
   ```bash
   python dataset.py
   ```
2. **Model Training**: Open and execute the cells in `Vizuara_AI_Labs_Small_Language_Model_Scratch_Final.ipynb` to tokenize the corpus (creating `train.bin` and `validation.bin`), initialize the model, and begin the training process.
